# Tencent Cloud Lighthouse Firewall

OctoBus service package for Tencent Cloud Lighthouse firewall operations. It uses Tencent Cloud API 3.0 TC3-HMAC-SHA256 signing and calls `lighthouse.tencentcloudapi.com` with API version `2020-03-24`.

## Supported Product and Version

- Product: Tencent Cloud Lighthouse / 轻量应用服务器
- API version: `2020-03-24`
- Endpoint: `https://lighthouse.tencentcloudapi.com`
- Verified interface scope: firewall rule list, create, delete, modify, firewall template apply, and IP block/unblock wrappers.

## Authentication

Use Tencent Cloud CAM credentials with Lighthouse firewall permissions.

`secret.schema.json`:

```json
{
  "secretId": "AKID...",
  "secretKey": "***",
  "token": "optional-sts-token"
}
```

`config.schema.json`:

```json
{
  "region": "ap-guangzhou",
  "endpoint": "https://lighthouse.tencentcloudapi.com",
  "timeoutMs": 5000
}
```

Request-level `credential` and `region` override instance secret/config values.

## Methods

- `ListFirewallRules`: maps to `DescribeFirewallRules`; input `instance_id`, optional `offset`, `limit`; output `total_count`, normalized `rules`, `request_id`, `raw`.
- `CreateFirewallRules`: maps to `CreateFirewallRules`; input `instance_id` and `rules[]` with `protocol`, `port`, `cidr_block`, `action`, optional `firewall_rule_description`.
- `DeleteFirewallRules`: maps to `DeleteFirewallRules`; deletes rules matching the same rule tuple.
- `ModifyFirewallRules`: maps to `ModifyFirewallRules`; input `rules[]` pairs of `firewall_rule` and `new_firewall_rule`.
- `ApplyFirewallTemplate`: maps to `ApplyFirewallTemplate`; input `template_id` and `instance_ids[]`.
- `BlockIP`: creates `DROP` firewall rules for `source_ips[]`; defaults `protocol=TCP`, `port=ALL`, `firewall_rule_description=octobus-block-ip`.
- `UnblockIP`: deletes the same `DROP` firewall rules generated by `BlockIP`; use the same protocol, port, and description to keep rollback deterministic.

## Risk Boundary

- Write operations change cloud firewall policy and may interrupt business traffic.
- `BlockIP` and `UnblockIP` are idempotent only to the extent supported by Tencent Cloud API duplicate/missing-rule handling; callers should record `source_ips`, `protocol`, `port`, `description`, `instance_id`, `region`, `request_id`, and operator/task ID for audit.
- Use least-privilege CAM policy scoped to Lighthouse firewall APIs and target regions.
- Do not store real `secretId`, `secretKey`, STS token, production IPs, screenshots, or sensitive business data in code, tests, docs, or PR text.

## Write Operation Semantics

- Defaults: `BlockIP` and `UnblockIP` default to `protocol=TCP`, `port=ALL`, `firewall_rule_description=octobus-block-ip`.
- Idempotency: Tencent Cloud rejects or ignores some duplicate/missing firewall rule tuples depending on upstream behavior. For deterministic automation, callers should query `ListFirewallRules` before and after writes.
- Rollback: rollback a `BlockIP` operation by calling `UnblockIP` with the same `instance_id`, `source_ips`, `protocol`, `port`, and `firewall_rule_description`; rollback a generic `CreateFirewallRules` call by calling `DeleteFirewallRules` with the same rule tuple.
- Audit fields: persist upstream `request_id`, OctoBus capset/instance, operator or workflow ID, `region`, `instance_id`, `protocol`, `port`, `cidr_block`, `action`, and `firewall_rule_description`.

## Suggested Capset

- Read-only: `ListFirewallRules`
- Firewall write: `CreateFirewallRules`, `DeleteFirewallRules`, `ModifyFirewallRules`, `ApplyFirewallTemplate`
- SOAR block/unblock: `BlockIP`, `UnblockIP`

## Verification Notes

- Import: `octobus service import tencent-lighthouse-firewall-test /path/to/tencent__lighthouse-firewall --reinstall`.
- Instance: `octobus instance create lighthouse-fw-real --service tencent-lighthouse-firewall-test --config config.json --secret secret.json`.
- Capset: `octobus capset create lighthouse-fw-real` and `octobus capset add-instance lighthouse-fw-real lighthouse-fw-real`.
- Core call: `CreateFirewallRules` and `BlockIP` were verified against Tencent Cloud Lighthouse API with a test rule, then checked with `DescribeFirewallRules`.
- Cleanup: remove test rules with `DeleteFirewallRules` or `UnblockIP` using the same protocol, port, CIDR, action, and description tuple.

## Known Limitations

- This package targets IPv4 firewall rule fields. IPv6 fields returned by Tencent Cloud are preserved only in `raw_json`.
- It does not resolve regions automatically; configure or pass the correct Tencent Cloud region.
- It does not merge or reorder conflicting upstream rules. Callers should inspect effective rule behavior in Tencent Cloud when both ACCEPT and DROP rules exist for the same port/CIDR.
